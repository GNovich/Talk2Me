## Feature 7 — Neutral→self voice migration

**What was done:**

- `talk2me/tts/voice_cloner.py`:
  - Added `neutral_seed_path` parameter to `__init__`; loads
    `assets/neutral_seed.wav` + `assets/neutral_seed_text.txt` at startup.
  - Added `has_neutral_seed` property.
  - `_load_neutral_seed()`: reads WAV, resamples to 24 kHz, stores text.
    Logs a warning (does not crash) if either file is missing.
  - `synthesize()` now accepts `migration_alpha: float = 1.0`. When
    `alpha < 1.0` and a neutral seed is loaded, the method concatenates
    `[neutral[:n_neutral] | participant[:n_participant]]` where proportions
    are determined by `alpha`. Neutral audio is prepended so F5-TTS
    "remembers" the participant voice at synthesis time.

- `talk2me/tts/reference_buffer.py`:
  - Added `migration_alphas` parameter (default `(0.2, 0.5, 0.8, 1.0)` for
    tiers 0–3) to `__init__`.
  - Added `migration_alpha(tier=None) -> float`: maps tier→alpha, clamps
    out-of-range inputs, defaults to current buffer tier.

- `config/exhibit.yaml`: added `engine.migration_alphas: [0.2, 0.5, 0.8, 1.0]`

- `assets/README.md` created to explain how to generate and place the neutral
  seed for exhibit operators.

**Deferred:**
- Audible A/B comparison of migration=true vs migration=false deferred to
  exhibit Mac. The neutral seed files (`assets/neutral_seed.wav` and
  `assets/neutral_seed_text.txt`) are not bundled — the operator must generate

## Feature 8 — Curated question bank

**What was done:**

- `questions/calibration.yaml` (6 questions, Phase 1): comfortable/
  observational prompts designed to harvest clean reference audio naturally.
- `questions/personal.yaml` (10 questions, Phase 2): lightly intrusive;
  tagged with `topic_hooks` for transcript-biased selection.
- `questions/confrontational.yaml` (12 questions, Phase 3): intense questions
  designed to sit in the body, delivered in the participant's own voice.
  Content/consent note added in the file header.

- `talk2me/engine/question_bank.py`:
  - `QuestionBank.load(questions_dir)`: reads all 3 YAML files, validates
    every entry has `id` and `text`, raises `ValueError` on malformed entries.
  - `QuestionBank.select(phase, used_ids, topic_hints)`: returns `(id, text)`
    of the next unused entry biased toward `topic_hints` overlap with
    `topic_hooks`. Cycles when a phase is exhausted (clears that phase's
    used_ids). Falls back to deterministic first-unused when no hint matches.
  - `QuestionBank.reload()`: re-reads YAML without restart.

- `talk2me/engine/__init__.py`: exports `QuestionBank`, `ConversationEngine`.

**Artist note:** all question content is first-draft. The files are
human-editable and hot-reloadable (`bank.reload()`) without restarting the
installation. The artist should tune tone and phrasing between gallery sessions.

## Feature 9 — Conversation state machine

**What was done:**

- `talk2me/engine/state_machine.py`:
  - `ConversationEngine.__init__(question_bank, calibration_turns, personal_turns,
    idle_timeout_seconds)`.
  - Phase transitions: Phase 1 for turns 1–calibration_turns, Phase 2 for
    turns calibration_turns+1 through calibration_turns+personal_turns,
    Phase 3 thereafter. Driven by `record_turn()` count.
  - `next_question() -> str`: calls `QuestionBank.select()` with topic hints
    extracted from the last 3 transcripts (words ≥ 4 chars, no NLP required).
  - `record_turn(transcript, question_asked)`: increments turn, updates phase
    for the next turn, appends transcript to history.
  - `should_reset() -> bool`: monotonic clock comparison against idle timeout.
  - `reset()`: clears all state for next participant.

- `talk2me/app.py`:
  - Imports and instantiates `QuestionBank` + `ConversationEngine` from config.
  - Replaces hard-coded `_PLACEHOLDER_QUESTION` with `engine.next_question()`.
  - Wires `migration_alpha` from `ref_buffer.migration_alpha()` when
    `engine.migration=true` and a neutral seed is loaded.
  - Wires idle-timeout reset: `engine.should_reset()` checked at loop top.
  - `Speaker` for TTS output is now created once (was re-created each turn).
  - Latency log line now includes `alpha=` and `phase=` fields.
  - Saved audio filenames now include phase: `t01_p1_output_TIMESTAMP.wav`.
  - `engine.record_turn()` called after successful playback.
