## Feature 13 — Telemetry, logging & latency dashboard

**Files changed:**
- `talk2me/telemetry.py` — new
- `talk2me/app.py` — wired `TelemetryLogger` into `run_loop()` and `supervisor_loop()`; added `--report` CLI argument
- `tests/test_telemetry.py` — 13 new tests

**What was built:**
- `TelemetryLogger` in `talk2me/telemetry.py`:
  - `log_turn(turn, phase, tier, alpha, stt_s, tts_s, play_s, total_s)` — appends
    one JSON line per completed turn to `logs/telemetry_YYYY-MM-DD.jsonl`
    (latency + phase metadata only; zero participant data).
  - Returns a one-line health banner: `[OK]` (≤3 s), `[SLOW]` (3–5 s), `[!!]` (>5 s).
  - `session_summary(date_str)` — reads the JSONL file and returns per-stage
    averages + count of over-budget turns.
  - `print_report(date_str)` — prints a human-readable per-turn table + session
    average; gracefully handles missing or empty files.
- `app.py` changes:
  - In kiosk mode: `TelemetryLogger` created once in `supervisor_loop()` and
    passed down to `run_loop()` as `telemetry_logger=`.
  - In dev mode: `TelemetryLogger` created in `main()` before calling `run_loop()`.
  - After each completed turn: `log_turn()` called; banner printed to stdout.
  - `talk2me --report YYYY-MM-DD` prints the full report and exits.

**Verification:**
- 13 unit tests pass (JSON validity, average computation, empty-file graceful
  handling, banner thresholds, `print_report` output).
- Functional: `TelemetryLogger` verified to produce correct JSONL output and
  compute correct averages through unit tests.  Live health banner output is
  stdout-observable on any run.
- On-hardware latency budget verification remains deferred to exhibit Mac.
