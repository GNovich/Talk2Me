## Feature 10 — Optional LLM-adaptive question selection

**Files changed:**
- `talk2me/engine/llm_adapter.py` (new)
- `talk2me/engine/state_machine.py` (wired)
- `talk2me/app.py` (wired)
- `config/exhibit.yaml` (added `engine.llm_model`)

**What was done:**
- Created `LLMAdapter` in `talk2me/engine/llm_adapter.py`. Wraps `mlx_lm.load`
  and `mlx_lm.generate` (imported lazily so the module never loads unless the
  flag is set). Default model: `mlx-community/Llama-3.2-3B-Instruct-4bit`.
- Tightly constrained system prompt: preserve tone, intensity, and phase; forbid
  topic invention; return only the revised question text.
- Drift guard: fall back to original question if output is empty, lacks a
  question mark, or is more than 3× longer than the input.
- `LLMAdapter.warm()` runs a throwaway call at startup for JIT compilation.
- `ConversationEngine.__init__` accepts `llm_adapter=None`; when present,
  `next_question()` calls `adapter.personalize()` and logs both original and
  adapted text.
- In `app.py`, the adapter is loaded only when `engine.llm: true`. Warms at
  startup alongside Whisper and F5-TTS.

**Deferred:** On-hardware latency verification (target: LLM call ≤1 s on M2/M3;
budget is ~0.3–0.5 s for 3B 4-bit). Concurrent LLM+TTS warm-up optimization
deferred as a future micro-optimization.

## Feature 11 — Kiosk runtime & session lifecycle

**Files changed:**
- `talk2me/app.py` (outer supervisor loop, signal handlers, standby mode)
- `scripts/com.talk2me.exhibit.plist` (new)
- `scripts/install_launchd.sh` (new)
- `logs/` directory created

**What was done:**
- `supervisor_loop()` in `app.py`: outer restart loop with `_MAX_RESTARTS=3`
  consecutive-crash guard, then standby. After standby, restarts on SIGUSR1.
- Signal handlers installed at startup: SIGUSR1 → session reset, SIGUSR2 →
  panic/purge, SIGTERM → clean shutdown.
- `_setup_logging(kiosk, project_root)`: in kiosk mode writes to
  `logs/session_YYYY-MM-DD.log` and routes unhandled exceptions to logging
  (suppresses raw tracebacks on stdout).
- `_standby_loop()` / `_standby()`: plays a 440 Hz sine pulse at ~0.02
  amplitude every 30 s as an ambient readiness signal; exits on SIGUSR1 or
  SIGTERM.
- `launchd` plist at `scripts/com.talk2me.exhibit.plist`:
  `KeepAlive=true`, `RunAtLoad=true`, `ThrottleInterval=10`.
  Valid XML confirmed by `plutil -lint`.
- `scripts/install_launchd.sh`: attendant-facing install helper with usage docs.
- `--kiosk` flag added to CLI: routes to `supervisor_loop()`.
- `config/exhibit.yaml`: added `kiosk: false` toggle.

**Attendant reset commands (documented in plist and install script):**
```
kill -USR1 $(pgrep -f "talk2me --kiosk")   # session reset
kill -USR2 $(pgrep -f "talk2me --kiosk")   # panic / purge
sudo launchctl stop com.talk2me.exhibit     # clean shutdown
```

**Deferred:** On-hardware launchd install and boot test. Ambient tone volume
is tunable but not yet exposed in `exhibit.yaml` (minor; add if exhibit needs it).

## Feature 12 — Consent, safety & privacy layer

**Files changed:**
- `talk2me/app.py` (consent_gate, _purge_session, _log_transcript,
  _assert_no_network_egress)
- `config/exhibit.yaml` (added `privacy.save_transcripts: false`)
- `PRIVACY.md` (new)

**What was done:**
- `consent_gate(privacy_cfg, kiosk)`: blocks on ENTER before each session.
  Returns False on 'q'/'quit'/'no'/'exit' or EOFError — returns to standby.
- `_purge_session(ref_buffer, engine)`: calls `reset()` on both objects.
  Called on: normal session end, idle timeout reset, panic, attendant reset,
  KeyboardInterrupt, and in a `finally` block to cover all exit paths.
- Panic flow (SIGUSR2): checks `_panic_requested` at the top of the turn loop,
  immediately calls `_purge_session` and returns to standby.
- `_log_transcript(text, path)`: opt-in only; gated by
  `privacy.save_transcripts`. Default is `false`. When true, anonymized text
  (no timestamps, no names) appended to `logs/transcripts_YYYY-MM-DD.log`.
- `_assert_no_network_egress(cfg)`: startup check that verifies the configured
  STT model is present in the HuggingFace cache. Raises `RuntimeError` if not,
  blocking the supervisor before any participant session starts.
- `PRIVACY.md`: gallery-facing data-handling document covering what is captured,
  retention policy, opt-out mechanism, biometric data notice, and operator
  incident response instructions.

**Deferred:** On-hardware test of `_assert_no_network_egress` (requires the
exhibit Mac where network is disabled). Secure-delete (overwrite with zeros
before unlink) of transcript files on session end — deferred until opt-in
logging path is actually in use at an exhibit.
