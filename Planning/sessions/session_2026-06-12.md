## Feature 4 — Zero-shot voice cloning with F5-TTS-MLX

**File:** `talk2me/tts/voice_cloner.py`

- `VoiceCloner` class wrapping `f5_tts_mlx.cfm.F5TTS`
- API: `VoiceCloner.synthesize(text, reference_wav, reference_text, reference_sample_rate) -> np.ndarray`
- Lazy model load + JIT warmup on `warm()` or first `synthesize()` call; model
  cached for session lifetime (no reload per call)
- Polyphase resampling (`scipy.signal.resample_poly`) handles 16 kHz mic audio →
  24 kHz F5-TTS native rate internally; no caller contract on input sample rate
- RMS normalisation to F5-TTS training level (TARGET_RMS = 0.1) before synthesis
- Duration estimated from reference/generation character-count ratio
- `sample_rate` property returns 24000; `Speaker` must be instantiated at this rate
- `_resample()` helper exposed for testing
- Per-synthesis latency printed to stdout

**Deferred:** Audible synthesis verification requires exhibit Mac with downloaded
`lucasnewman/f5-tts-mlx` model weights. The bundled `f5_tts_mlx/tests/test_en_1_ref_short.wav`
is used for JIT warmup. Integration test marked `@pytest.mark.model`.

## Feature 5 — End-to-end single-turn loop

**File:** `talk2me/app.py`

- Full orchestration: `record_utterance → Transcriber.transcribe → ReferenceBuffer.push → VoiceCloner.synthesize → Speaker.play`
- Per-stage `time.perf_counter()` instrumentation; per-turn breakdown printed:
  `[STT: X.Xs | TTS: X.Xs | play: X.Xs | total: X.Xs]`
- Pre-warms Whisper and F5-TTS at startup before first participant turn
- Hard-coded placeholder question `"What brought you here today?"` (Feature 8
  will replace with the curated bank)
- Edge cases handled: empty transcript → re-prompt; no reference yet (first-turn
  quality failure) → skip synthesis; synthesis/hardware error → log + continue;
  KeyboardInterrupt → graceful shutdown
- Per-turn synthesis output saved to `saved_audio/t<nn>_output_<ts>.wav`
- `main()` entry point wired to `talk2me` CLI command

**Deferred:** Live end-to-end verification with real microphone and F5-TTS weights
requires exhibit Mac. On-hardware latency target is ≈2–3 s total round-trip;
regressions will be caught when Feature 13 (telemetry) is wired.

**Note:** `Speaker` for TTS output is instantiated at 24 kHz (F5-TTS native).
The mic `Microphone`/VAD pipeline remains at 16 kHz. This is intentional;
the two rates are decoupled by the resampling in `VoiceCloner.synthesize()`.

## Feature 6 — Rolling reference accumulation

**File:** `talk2me/tts/reference_buffer.py`

- `ReferenceBuffer` accumulates `(wav, TranscriptResult)` tuples from participant utterances
- Quality gate (per segment): RMS > 0.005, peak < 0.95, avg_logprob > −1.5, non-empty text
- Segments ranked by `avg_logprob` descending; best segments up to 30 s cap are
  concatenated; re-sorted to capture order for a natural reference clip
- `tier` property: 0 (no ref) → 1 (≥3 s) → 2 (≥9 s) → 3 (≥20 s)
- `best_reference()` result cached; invalidated on `push()`
- `reset()` wipes all accumulated audio; must be called at session end (Feature 11/12)
- Wired into `app.py`: after each accepted turn, push transcript + wav; next turn
  uses the updated best reference

**Verification (logic):** 14 unit tests covering all quality-rejection cases,
tier advancement, quality-ranked selection, max-reference cap, cache invalidation,
and reset. All pass without audio hardware.

**Deferred:** Audible verification of progressive voice sharpening across 6+ turns
requires exhibit Mac and model weights.
